# YOLO 資料夾合併工具 (YOLO Dataset Merger)

這是一個專門用於處理 YOLO 格式標註資料夾的合併工具。當您的資料集被拆分成多個以數字編號結尾的子資料夾（例如分段處理的影片影格）時，此工具可以依照順序將它們合併成一個完整的資料集，並自動處理重複檔名問題。

## 🌟 核心功能

* 前綴自動匹配：透過指定 Prefix（如 CGTD01_260310_075343），自動搜尋 origin 目錄下所有符合 {Prefix}_{Index} 格式的資料夾。

* 序號檢查機制：自動檢查資料夾編號是否連續，若發現編號缺漏會主動跳出彈窗詢問是否繼續。

* 自動重新命名：為了避免不同資料夾內有相同檔名（例如都是 frame_000001.jpg），合併時會自動將檔名修改為 {資料夾編號}_{原始檔名}。

* 完整資料保留：自動從第一個子資料夾複製 classes.txt，並合併所有的 images 與 labels。

* 直覺式 GUI 介面：提供圖形化介面，支援目錄瀏覽、即時執行日誌顯示。

## 📁 預期路徑結構

在使用此工具前，請確保您的資料存放符合以下結構：
```
Target Path (您的工作目錄)/
├── origin/                         # 存放原始分段資料夾
│   ├── CGTD01_260310_075343_1/    # 第一段
│   │   ├── images/
│   │   ├── labels/
│   │   └── classes.txt
│   ├── CGTD01_260310_075343_2/    # 第二段
│   │   └── ...
│   └── CGTD01_260310_075343_5/    # (假設缺了 3, 4，工具會警告)
└── results/                        # (執行後自動生成)
```

## 🛠️ 合併後的命名規則

工具會將檔案搬移至 results/{Prefix}_MERGED/ 下，檔名變更範例：

* 原檔案：CGTD01_..._1/images/frame_001.jpg -> 合併後：1_frame_001.jpg

* 原檔案：CGTD01_..._2/images/frame_001.jpg -> 合併後：2_frame_001.jpg

這確保了資料在合併後不會發生覆蓋，且依然能保有原本的順序性。

## 🚀 如何使用

1. 執行程式：啟動 Python 腳本或執行封裝後的 .exe 檔案。

2. 設定路徑：

  * Target Path：選擇包含 origin 資料夾的父目錄。

  * Prefix：輸入資料夾的共同前綴（不含底線與編號，例如 CGTD01_260310_075343）。

3. 點擊「開始合併」：

  * 工具會先掃描 origin 目錄。

  * 若編號不連續，會詢問您是否繼續。

4. 檢查結果：合併完成後，請至 results 資料夾查看合併後的結果。

## 📦 封裝建議 (PyInstaller)

若要將此工具打包成單一執行檔，請使用以下指令：
```bash 
pyinstaller --onefile --noconsole --clean --name "YOLO_Merger" merge_YOLO.py
```
